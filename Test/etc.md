- 소프트웨어 개발 후 테스트 가능한 최소 단위를 기준으로 하는 테스트 활동

### 단위 테스트 코드와 프로덕션 코드를 구분해야 되는 이유
- 테스트 코드와 프로덕션 코드가 하나의 클래스에 존재하면 → 기능이 늘어남에 따라 클래스의 크기가 커지고 → 복잡도가 증가한다
- 테스트 코드와 프로덕션 코드가 실제 서비스에 같이 배포된다 (TODO : 테스트 코드를 배포 시 제외할 수 있나보네?)
